## ⚠️ 破坏性变更 (BREAKING CHANGES)

### 1. Flash 分区表重新设计

Flash 分区布局进行了重构，**新版本 Web 固件必须重新上传**，旧版 FSBL 和 WiFi 固件版本不匹配时系统会弹出提示。

| 分区       | 旧版本 (v2.1.0)         | 新版本 (v2.3.0)          | 变更说明                    |
|-----------|------------------------|-------------------------|----------------------------|
| AI_1      | 5MB @ 0x70E00000       | 8MB @ 0x70900000        | 地址前移，大小翻倍（合并原 AI_DEFAULT） |
| AI_2      | 5MB @ 0x71300000       | 8MB @ 0x71100000        | 地址前移，大小翻倍（合并原 AI_3） |
| AI_DEFAULT | 5MB @ 0x70900000      | — 已移除 —              | 合并至 AI_1                 |
| AI_3      | 8MB @ 0x71800000       | — 已移除 —              | 合并至 AI_2                 |
| WEB       | — 不存在 —             | 1MB @ 0x71900000        | **新增** Web 独立分区         |
| WiFi FW   | 4MB @ 0x77C00000       | 3MB @ 0x71A00000        | 地址前移，大小调整             |
| LittleFS  | 64MB @ 0x72000000      | 32MB/96MB @ 0x71D00000  | 地址变更，支持 64M/128M Flash |
| Reserve2  | 28MB @ 0x76000000      | 4MB @ 0x73D00000/0x77D00000 | 地址和大小变更            |

### 2. PSRAM 内存布局变更

- **APP 固件 PSRAM 启动**: 上一版本已通过 `BOOT_IN_PSRAM` 支持 PSRAM 运行 APP，此版本**移除条件编译**，PSRAM 启动成为唯一方式。APP 可用空间从 4032K 扩大至 4096K
- **PSRAM SWAP 区域删除**: 位于 `0x903F0000` 的 64KB PSRAM SWAP 区域已删除
- **新增 SRAM_POOL 区域**: `0x34000000-0x341B1FFF` (1736K)，原 SRAM APP 空间转为内存池
- **UNCACHED 区域调整**: 起始地址从 `0x341A0000` 移至 `0x341B2000`

### 3. WiFi SDK 升级

- SiWG917 SDK 升级至 **v4.0.2 SDK**，配套 WiFi 固件 `SiWG917-B.2.15.5.0.0.2`
- 旧版 WiFi 固件（如 `2.14.5.x`）仍可正常使用，但建议升级以获得最新改进
- 新增 `EXPECTED_WIFI_VERSION` 编译期版本校验机制

## 新功能 (Features)

### WiFi 固件 OTA 升级
- 新增 WiFi (SiWG917) 固件的 OTA 升级支持，通过 Web 界面或系统设置上传并更新
- **两阶段升级流程**:
  1. **上传阶段**: 流式写入 Flash (`WIFI_FW_BASE`)，写入 `flash_header_t` + `.rps` 固件本体，OTA 包头仅用于 Web 端校验不写入 Flash
  2. **确认阶段**: 通过 `ota_upgrade_local_handler` 设置 NVS `wifi_mode=update` 标志，设备重启后由 `wifi_mode_process()` → `firmware_upgrade_from_flash()` 将 `.rps` 推送至 SiWG917 芯片
- **WiFi 版本检测与提示**: 系统同时读取 Flash 中存储的 WiFi 固件版本和芯片当前运行版本，在固件升级页面展示并自动检测版本不匹配
- **期望版本校验**: 编译时通过 `EXPECTED_WIFI_VERSION` 指定期望的 WiFi 固件版本，当 Flash 和运行版本一致但与期望值不同时，Web UI 将弹出引导升级对话框
- **升级等待页**: WiFi 固件推送至芯片耗时较长（约 1-3 分钟），前端在重启后轮询设备状态，超时后跳转至 LED 指引等待页 (`/upgrade-waiting`)
- `firmware_upgrade_from_flash()` 增加 `storage_lock/unlock` 保护，避免与文件系统并发访问冲突；增加各错误路径的资源释放
- 新增 `wifi_mark_update_pending()` 接口：设置 NVS 标志但不立即复位，允许 HTTP 响应正常返回
- 新增 `wifi_get_running_version()` 和 `wifi_get_flash_version()` 接口：分别从芯片和 Flash 读取 WiFi 固件版本

### Web Recovery 恢复系统
- 新增 `web_recovery` 模块：Web 固件分区为空或损坏时自动进入内置恢复页面
- 恢复页为编译在 APP 固件中的自包含 HTML 页面（无外部资源依赖），提供 Web 固件上传功能
- Web 固件上传成功后自动调用 `web_recovery_reload_assets()`，从 Flash 重载 assets 并退出恢复模式，**无需重启设备**

### AI 分区重构
- `FIRMWARE_DEFAULT_AI` 重命名为 `FIRMWARE_AI_2`，保留 `FIRMWARE_AI_1`，支持 AI 模型 A/B 双槽切换
- AI 分区大小从 5MB 增大至 8MB，适配更大的模型文件

### Web 端固件版本管理增强
- 固件升级页面大改版：
  - 导入固件对话框支持 APP / Web / AI 模型 / 设备配置文件
  - 高级选项菜单提供独立的 FSBL 升级和 WiFi 升级入口
  - 新增 WiFi 固件升级独立页面 (`/import-wifi`)
  - 固件信息页展示全组件版本：APP / Web / AI 模型 / FSBL / WakeCore / WiFi
- **FSBL 期望版本校验**: 编译时通过 `EXPECTED_FSBL_VERSION` 指定期望 FSBL 版本，Web UI 自动检测并弹出升级引导
- **WiFi 期望版本校验**: 同上，通过 `EXPECTED_WIFI_VERSION` 校验

### HaLow (802.11ah) 支持
- HaLow Wi-Fi 完整集成：快速连接、运行时射频同步、重连 IP 修复
- 扩展 RegDomain 国家码支持（端到端）
- PKA 加密硬件卸载
- IP 配置持久化存储
- SPI6 时钟自适应、放宽 EU 占空比限制
- Web UI 支持 HaLow 网络配置 (halow-network.tsx)

### AI 与模型
- YOLOv8-Nano / YOLO11n-Nano 人体姿态估计 (4 关键点)
- 指针仪表读数: YOLOv8-pose 4 关键点 → 百分比读数转换
- 实例分割 (ISEG): YOLOv8 模型集成
- BlazeFace 人脸检测: 128×128 分辨率
- 可配置 AI 推理频率
- 连续 AI 结果遥测：支持 CBOR 精简格式、仅元数据模式
- SPE 结果处理：`overlay_results` 门控

### 抓拍上传管道
- 新增抓拍-上传完整管道，Web UI 可配置抓拍任务计划
- 支持定时抓拍和手动触发

### 存储与文件管理
- Web 端文件管理：支持 Flash 和 SD 卡文件的浏览、上传、删除
- 文件操作注册与 LFS 挂载解耦，提升启动健壮性

### 硬件支持
- 双芯 NOR Flash 自适应驱动: 自动识别 MX66 (1Gb) 和 GD55 (1Gb/512Mb)
- 支持 64MB 和 128MB Flash 容量 (`BOARD_FLASH_SIZE`)
- ISP 灰阶模式：重启生效

### 网络
- 4G 手动 PLMN 运营商选择
- eth_wan 默认接口处理
- Modem 初始化和电源管理修复

## 修复 (Bug Fixes)

- **Web 端 use-after-free**: 修复 HaLow 区域设置处理器中的内存安全问题
- **JPEG 缓冲区混叠**: 修复抓拍和 JPEG 编码器中的缓冲区混叠问题
- **eth_wan/halow 初始化**: 修复无蜂窝模块时的 eth_wan 默认值和 halow 初始化
- **密码学清理**: 移除未使用的 si91x PSA 加密组件，整理 mbedtls PKA 移植
- **错误消息乱码**: 修复错误消息字符串显示异常
- **补光灯 PWM 亮度**: 修复亮度缩放问题
- **Flash 映射读取保护**: 增加读取保护逻辑
- **RegDomain 选择修复**: 按 regdomain 正确选择 BCF 并暴露元数据
- **WiFi 固件升级资源保护**: `firmware_upgrade_from_flash()` 增加 `storage_lock/unlock`，防止与文件系统并发冲突
- **WiFi 升级后 SDK 初始化**: `firmware_upgrade_via_xmodem` 完成后调用 `sl_net_deinit` 正确释放网络栈

## 技术改进 (Improvements)

- APP PSRAM 启动成为唯一模式（移除 `BOOT_IN_PSRAM` 条件编译），APP 可用空间扩大至 4096K
- Flash 分区 SWAP 区域从 Flash 移至 PSRAM 区域首部
- CRC32 计算从硬件 CRC (`HAL_CRC_Calculate`) 切换至软件实现 (`CRC_Calculate`)，避免硬件 CRC 外设的字节序问题
- OTA 上传完成后 Web 固件自动重载（无需设备重启），升级体验更流畅
- 移除旧版 si91x WiFi SDK 遗留代码
