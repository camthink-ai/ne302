# NE301 v2.3.0 OTA Update Guide

> **English** | [中文](./NE301_v2.3.0_OTA_Update_Guide_cn.md)

> **Applicable Scenario**: Upgrading from NE301 v2.1.0 (commit `0198d97` or earlier) to v2.3.0
>
> **Prerequisites**: The device is running the old firmware normally. All OTA firmware files are ready on your PC.
>
> ⚠️ **IMPORTANT**: This release introduces a redesigned Flash partition table with a **brand-new independent Web partition** (which did not exist in previous versions). The upgrade **must follow the exact steps and sequence** in this guide. Skipping steps or reordering them will result in the Web interface becoming inaccessible.

## Upgrade Overview

v2.3.0 moves the Web assets into a dedicated Flash partition (`WEB_BASE = 0x71900000`). After upgrading, the old Web assets are no longer accessible — the device will automatically enter the built-in Web Recovery mode. The upgrade must be performed in the following sequence:

```
Old Version v2.1.0
    │
    ├── 1. Upload new APP firmware → Device reboots → APP upgrade complete (new partition table active)
    │
    ├── 2. Connect to device WiFi → Open browser → Enter Web Recovery mode
    │     (Reason: Web partition address changed; old Web assets are no longer accessible)
    │
    ├── 3. On Recovery page → Upload new Web OTA firmware → Page auto-recovers
    │     (Web assets reload automatically after upload — no reboot needed)
    │
    ├── 4. Enter normal UI → System prompts FSBL version mismatch → Follow guided upgrade
    │     (If no prompt: System Settings → Firmware Upgrade → Import → Advanced → FSBL Upgrade)
    │
    ├── 5. Reconnect WiFi → System prompts WiFi version mismatch → Follow guided upgrade
    │     (If no prompt: System Settings → Firmware Upgrade → Import → Advanced → WiFi Upgrade)
    │     ⏳ WiFi firmware push to chip takes ~1–3 minutes — please be patient
    │
    └── 6. Upgrade complete → Verify version information
```

### Why Is This Process Necessary?

| Change                              | Impact                                                            |
|-------------------------------------|-------------------------------------------------------------------|
| Web partition now independent (0x71900000) | Old APP has no concept of this partition; Web assets are lost → triggers Recovery |
| FSBL expected version check (1.0.3.0)      | APP is built with `EXPECTED_FSBL_VERSION=1.0.3.0`; prompts on mismatch |
| WiFi partition relocated + SDK upgraded    | APP is built with `EXPECTED_WIFI_VERSION=2.15.5.2`; prompts on mismatch |

## Preparation

### Required OTA Firmware Files

| Filename                                   | Type   | Notes                            |
|-------------------------------------------|-------|----------------------------------|
| `ne301_App_signed_v2.3.0.133_pkg.bin`     | APP   | Main application OTA package     |
| `ne301_Web_v1.5.0.0_pkg.bin`              | Web   | Web frontend assets OTA package  |
| `ne301_FSBL_signed_v1.0.3.0_pkg.bin`      | FSBL  | First Stage Boot Loader OTA pkg  |
| `ne301_Wifi_flash_v2.15.5.2_pkg.bin`      | WiFi  | SiWG917 wireless firmware OTA pkg |

### Operating Environment

- A PC or phone that can connect to the device's WiFi hotspot
- Chrome / Edge browser recommended (use incognito mode if you encounter caching issues)

## Step 1: Upload the New APP Firmware

> **Goal**: Write the v2.3.0 APP firmware to Flash. After reboot, the new partition table and features take effect.

### Steps

1. Ensure the device is running the old firmware normally. Connect your PC to the device's WiFi hotspot.
2. Open a browser and navigate to the device's Web management interface.
3. Go to **System Settings** → **Firmware Upgrade** → Click **Import Firmware**

![Import firmware entry](images/step1_import_entry.png)

4. In the Import Firmware dialog, click the **APP File** area to select a file.

![Select APP firmware](images/step1_select_app.png)

5. Select `ne301_App_signed_v2.3.0.133_pkg.bin`.
6. Wait for the upload to complete (progress is shown during upload).
7. Click the **Confirm Burn** button.

![Confirm burn](images/step1_confirm_burn.png)

8. The device will reboot automatically. A waiting screen will be displayed.

### Expected Result

- The device reboots automatically.
- After about 5–15 seconds, search for the device's WiFi hotspot again.

---

## Step 2: Web Firmware Recovery

> **Goal**: Enter Web Recovery mode in preparation for uploading the new Web firmware.

### Steps

1. Reconnect your PC to the device's WiFi hotspot.
2. Open a browser and enter the device's management address (or refresh the previous page).

3. You should see the **Web Recovery Mode** recovery page:

![Web Recovery page](images/step2_recovery_page.png)

> The page displays: "The device could not load a valid web firmware (the partition address may have changed after an upgrade, or the firmware was not flashed)."

4. **If the recovery page does not appear**, try:
   - Opening a new browser tab (incognito/private mode recommended).
   - Clearing the browser cache and refreshing (Ctrl+Shift+Del).
   - Switching to a different browser (Chrome → Edge → Firefox).

---

## Step 3: Upload the New Web OTA Firmware

> **Goal**: Upload the new Web firmware through the Recovery page to restore the normal Web UI.

### Steps

1. On the Web Recovery page, click the file selection area.
2. Select `ne301_Web_v1.5.0.0_pkg.bin`.

![Select Web firmware](images/step3_select_web.png)

3. Click the **Upload & Recover** button.
4. The progress bar will show real-time upload progress.

![Web upload progress](images/step3_upload_progress.png)

5. After a successful upload, the page displays "Recovery successful, reloading interface..." and **automatically refreshes** to the normal Web management interface after ~1.5 seconds.

![Recovery success](images/step3_recovery_success.png)

6. **If the page does not auto-refresh**: manually refresh the browser page (F5).

### Expected Result

- The normal Web management interface loads successfully.
- All function menus are visible.

---

## Step 4: FSBL Firmware Upgrade

> **Goal**: Update the FSBL to `1.0.3.0` and clear the version mismatch warning.

### Automatic Prompt

1. In the Web management interface, go to **System Settings** → **Firmware Upgrade**.

2. Check the **FSBL Version** row — if the current version does not match the expected version (`1.0.3.0`), a **prompt icon** appears on the right:

![FSBL version mismatch prompt](images/step4_fsbl_mismatch.png)

3. Click the icon to open a dialog showing the current version vs. the expected version:

![FSBL mismatch dialog](images/step4_fsbl_dialog.png)

4. Click **Upgrade Now** to jump to the FSBL upgrade page.

### Manual Entry

If no automatic prompt appears: **System Settings** → **Firmware Upgrade** → **Import Firmware** → **Advanced Options** (bottom-left) → **FSBL Firmware Upgrade**

![Advanced options menu](images/step4_advanced_menu.png)

### Upload and Complete

1. On the FSBL upgrade page, click the file selection area.
2. Select `ne301_FSBL_signed_v1.0.3.0_pkg.bin`.

![Select FSBL firmware](images/step4_select_fsbl_file.png)

3. Click **Confirm Burn**. The device will reboot to complete the FSBL upgrade.

> ⚠️ **Note**: After clicking **Confirm Burn**, the page may appear unresponsive. However, the device has actually completed the FSBL upgrade in the background and is rebooting. Wait about 5–10 seconds, reconnect to the device's WiFi, and go to **System Settings → Firmware Upgrade** to verify the FSBL version has been updated.

### Expected Result

- After the device reboots, reconnect WiFi and go to Web → System Settings → Firmware Upgrade → FSBL version displays `1.0.3.0` (prompt icon disappears).

## Step 5: WiFi Firmware Upgrade

> **Goal**: Update the SiWG917 wireless chip firmware to `2.15.5.2` (bundled with SDK v4.0.2).

⚠️ **Note**: Pushing the WiFi firmware to the SiWG917 chip's internal storage takes approximately **1–3 minutes** — the longest step in the entire process. Please be patient and **do not power off the device** during this step.

### Automatic Prompt

1. Reconnect your PC to the device's WiFi, enter the Web management interface → **System Settings** → **Firmware Upgrade**.

2. Check the **WiFi Version** row:
   - **Flash version** ≠ **Running version**: a prompt icon appears.
   - Both match but ≠ **Expected version** (`2.15.5.2`): a prompt icon appears.

![WiFi version prompt](images/step5_wifi_mismatch.png)

3. Click the prompt icon → **Upgrade Now**.

### Manual Entry

**System Settings** → **Firmware Upgrade** → **Import Firmware** → **Advanced Options** → **WiFi Firmware Upgrade**

### Steps

1. On the WiFi upgrade page, click the file selection area.
2. Select `ne301_Wifi_flash_v2.15.5.2_pkg.bin`.

![Select WiFi firmware](images/step5_select_wifi_file.png)

3. After the file upload completes, click **Confirm Upgrade**.
4. A confirmation dialog appears. Review and click **Confirm**.

![WiFi confirm upgrade](images/step5_confirm_wifi.png)

5. The device reboots and begins pushing the WiFi firmware.

### Wait for Completion

What happens after reboot:
- The LED **blinks rapidly** (WiFi firmware is being pushed to the SiWG917 chip).
- The WiFi hotspot will disconnect and needs to be manually reconnected after the update completes.
- The PC frontend polls the device (every 5 seconds, up to 60 seconds) to detect when it is back online.

If the frontend cannot detect the device, the page redirects to the **LED Guidance waiting page** (`/upgrade-waiting`):

![LED waiting page](images/step5_led_waiting.png)

Use the LED to determine status:
- **LED solid on** → Upgrade complete.
- **LED blinking** → Upgrade in progress.
- **LED off** → Abnormal or device entered sleep mode. Try pressing and holding the button for 2+ seconds to wake.

### Expected Result

- On the Firmware Upgrade page, WiFi version displays `2.15.5.2` (Flash and running versions match).

---

## Step 6: Verify the Upgrade

1. Go to **System Settings** → **Firmware Upgrade**.
2. Verify the following version information:

| Component        | Expected Version | Status          |
|-----------------|-----------------|-----------------|
| APP Version      | 2.3.0.x         | ✓               |
| Web Version      | 1.5.0.0         | ✓               |
| FSBL Version     | 1.0.3.0         | ✓ No prompt icon |
| WiFi Version     | 2.15.5.2        | ✓ No prompt icon |
| AI Model Version | 2.0.0.0         | ✓               |
| WakeCore         | 0.2.7.5         | ✓               |

![Verify versions](images/step6_verify_versions.png)

3. If all versions are correct and there are no prompt icons → OTA upgrade is complete.

## FAQ

### Q: What if the device loses power during the upgrade?

- **Power loss during APP upgrade**: The A/B partition protection mechanism keeps the old slot firmware intact. The device will boot from the old slot on next power-up. The new slot is marked `UNBOOTABLE` and the system rolls back automatically.
- **Power loss during FSBL upgrade**: FSBL corruption will prevent the device from booting. Re-flash via JTAG/SWD.
- **Power loss during WiFi upgrade**: If power is lost while `.rps` is being pushed to the chip, WiFi may function abnormally after reboot. Re-enter the WiFi upgrade page and upload again.

### Q: The Web Recovery page does not appear?

- Open the browser in incognito/private mode and re-enter the device IP.
- Clear the browser cache.
- Try a different browser.
- Verify that your PC is connected to the device's WiFi hotspot.

### Q: The FSBL/WiFi upgrade prompt does not appear?

Manual paths:
- **FSBL**: System Settings → Firmware Upgrade → Import Firmware → Advanced Options → FSBL Firmware Upgrade
- **WiFi**: System Settings → Firmware Upgrade → Import Firmware → Advanced Options → WiFi Firmware Upgrade

### Q: Cannot connect to the device after WiFi firmware upgrade?

1. Wait at least 3 minutes and observe the device LED.
2. LED is solid on but no WiFi hotspot → try rebooting the device.
3. Still not recovered → re-flash the full firmware package via JTAG/SWD.

### Q: Does the old WiFi firmware still work? Can I skip the WiFi upgrade?

**Yes, you can continue using it**. The old WiFi firmware (e.g., `2.14.5.x`) remains compatible with SDK v4.0.2. However, the system will show a prompt icon on the version check page based on `EXPECTED_WIFI_VERSION=2.15.5.2`. Upgrading is **recommended** for the latest improvements.

### Q: Can I skip the FSBL upgrade?

**Not recommended**. APP v2.3.0 is built with `EXPECTED_FSBL_VERSION=1.0.3.0`. The firmware info page will persistently display a warning if the versions do not match.
